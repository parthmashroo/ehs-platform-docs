Risk Management · Duck Creek

Tell me about a high-risk upgrade or migration on a production system — how did you manage the risk?

Think about it first — then click to reveal

Situation
Duck Creek Billing — document generation pipeline running Aspose v14 (9 major versions behind) and PDF4NET v3. Both outdated, both used for customer-facing invoice, statement, and write-off document rendering. Major version jumps — high risk of rendering regressions.
Action
Started with full impact analysis — mapped every document type rendered through both libraries. Upgraded Aspose v14 → v23 and PDF4NET v3 → v10 incrementally, building regression coverage for each document type: invoices, statements, write-offs. Verified visual output parity, not just no-crash.
Result
Both libraries successfully upgraded with full regression coverage in place. No rendering regressions reached production. Document pipeline modernized and technically current.
