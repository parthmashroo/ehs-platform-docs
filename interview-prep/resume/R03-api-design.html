API Design · Duck Creek

Tell me about an API you designed that had a meaningful performance impact on the system.

Think about it first — then click to reveal

Situation
Duck Creek Billing — existing session patterns were loading entire policy datasets. For high-volume accounts this meant pulling ~100,000 policies into memory just to display a filtered view.
Action
Designed and shipped 3 new server-side API endpoints: a date-range-filtered BillItem + PolicyTerm join that replaced full session loads, and an installment-schedule filter that pushed predicate logic to the database instead of filtering in application memory.
Result
Eliminated the pattern of pulling 100K policies for high-volume accounts. Queries now return only what the UI needs — smaller payloads, faster responses, lower memory pressure.
